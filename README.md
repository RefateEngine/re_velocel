# velocel

`velocel` は、Minecraftのデータパック開発において、エンティティの「絶対速度」「絶対加速度」、および「視点基準の移動速度・加速度（前後・左右）」と「運動方向の角度（Yaw/Pitch）」をリアルタイムで追跡・計算するための軽量・高パフォーマンスな数学ライブラリです。

## 使い方
追跡したいエンティティを実行者にして関数を実行します
```
function re_velocel:track/
```
その後エンティティのスコアボードに値が入ります

### 1. 座標データ
| スコア名 | 表示名 | 倍率 | 説明 |
| :--- | :--- | :---: | :--- |
| `re_velocel_posX` | X座標 | 1000倍 | エンティティの現在のワールドX座標 |
| `re_velocel_posY` | Y座標 | 1000倍 | エンティティの現在のワールドY座標 |
| `re_velocel_posZ` | Z座標 | 1000倍 | エンティティの現在のワールドZ座標 |

### 2. 絶対運動データ（ワールド座標基準）
| スコア名 | 表示名 | 倍率 | 説明 |
| :--- | :--- | :---: | :--- |
| `re_velocel_velX` | X速度 | 1000倍 | ワールドX軸方向の速度（1tickあたりの移動量） |
| `re_velocel_velY` | Y速度 | 1000倍 | ワールドY軸方向の速度（1tickあたりの移動量） |
| `re_velocel_velZ` | Z速度 | 1000倍 | ワールドZ軸方向の速度（1tickあたりの移動量） |
| `re_velocel_accelX` | X加速度 | 1000倍 | ワールドX軸方向の加速度（1tickあたりの速度変化量） |
| `re_velocel_accelY` | Y加速度 | 1000倍 | ワールドY軸方向の加速度（1tickあたりの速度変化量） |
| `re_velocel_accelZ` | Z加速度 | 1000倍 | ワールドZ軸方向の加速度（1tickあたりの速度変化量） |
| `re_velocel_velTotal` | 速度合計絶対値 | 1000倍 | 3軸の速度ベクトルの絶対値合計（マンハッタン距離） |
| `re_velocel_accelTotal` | 加速度合計絶対値 | 1000倍 | 3軸の加速度ベクトルの絶対値合計（マンハッタン距離） |

### 3. 相対運動データ（エンティティの視点基準）
| スコア名 | 表示名 | 倍率 | 説明 |
| :--- | :--- | :---: | :--- |
| `re_velocel_velFB` | 前後速度 | 1000倍 | 視点基準の速度（プラス：前方 / マイナス：後方） |
| `re_velocel_velRL` | 左右速度 | 1000倍 | 視点基準の速度（プラス：右平行 / マイナス：左平行） |
| `re_velocel_accelFB` | 前後加速度 | 1000倍 | 視点基準の加速度（プラス：前方加速 / マイナス：後方加速・ブレーキ） |
| `re_velocel_accelRL` | 左右加速度 | 1000倍 | 視点基準の加速度（プラス：右加速 / マイナス：左加速） |

### 4. 最も強い運動方向の角度（角度データ）
*※力がデッドゾーン（35未満）の時は、最後に動いていた方向の角度が維持されます。*
| スコア名 | 表示名 | 倍率 | 説明 |
| :--- | :--- | :---: | :--- |
| `re_velocel_velAngleYaw` | 速度角度Yaw | **10倍** | 速度ベクトルが示す水平方向の角度（`-1800` 〜 `1800`） |
| `re_velocel_velAnglePitch` | 速度角度Pitch | **10倍** | 速度ベクトルが示す上下方向の角度（`-900` 〜 `900`） |
| `re_velocel_accelAngleYaw` | 加速度角度Yaw | **10倍** | 加速度ベクトルが示す水平方向の力のかかっている角度 |
| `re_velocel_accelAnglePitch` | 加速度角度Pitch | **10倍** | 加速度ベクトルが示す上下方向の力のかかっている角度 |

## ⚠️ 注意事項
**固定UUIDの使用** : 計算用の2体のマーカーエンティティには固定のUUID（55ea1109-..., 7bd848b4-...）を使用しています。万が一消えてしまっても計算前に再召喚されますが特殊な非同期処理やチャンク外へのロード/アンロードを跨ぐような処理に組み込むと、NBTの競合が発生する可能性があります。
**テレポート（瞬間移動）時の異常値** : tp コマンド等でエンティティを長距離テレポートさせた場合、1tick前との座標差分が巨大になり、速度や加速度が異常な数値として算出されます。テレポートを行う際は、直後の1tickは計算結果を利用しないなどの対策を推奨します。対策をする場合は速度合計が一定以上の場合などで条件付けするのがおすすめです。
**誤差** : 角度データは移動量がごくわずかな場合1度から2度程度の誤差が生じます。気になる場合はデッドゾーンを調整してください
